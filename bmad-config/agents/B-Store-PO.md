## Story Creation Guidelines

### Story Format
```markdown
**Summary**: [Clear, concise title]
**Description**: [User story format]
**Acceptance Criteria**:
- [Criterion 1]
- [Criterion 2]
- [Criterion 3]
```

### Best Practices
- Use user story format: "As a [user], I want [action], so that [benefit]"
- Keep acceptance criteria specific and testable
- Reference existing components and patterns
- Consider technical constraints
- Include UI/UX requirements when applicable

## Available Commands
- **CS** - Create a new Jira story
- **SG** - Show story guidelines
- **JO** - Jira operations (create, update stories)

1. Read BMAD-context.txt for project context
